# System Meaning Audit

## Strong Systems
| System | Why It Works | Risk |
| --- | --- | --- |
| Core matching | Clear, immediate, skill-based. | Can become visually busy as systems stack. |
| Traps | Create risk and recent softlock work improved fairness. | Need stronger "sprung" visual and card copy. |
| Exits/levers/locks | Create floor goals beyond clearing pairs. | Can feel like chores if rewards are weak. |
| Route choices | Give between-floor direction. | Needs more visible connection to next board. |
| Destroy/peek/shuffle | Clear utility verbs. | Button parity and perfect-rule copy must stay honest. |

## Weak Or Random-Feeling Systems
| System | Current Weakness | Theory Need |
| --- | --- | --- |
| Relics | Many are charges, caps, or generic bonuses. | Redesign around build-defining moments and visual identity. |
| Mutators | Several alter rules but lack strong floor presentation. | Give each mutator a board tell, floor identity, and counterplay. |
| Talents/traits | Not a coherent named system yet. | Decide whether to create a real layer or reframe existing layers. |
| Findables | Optional pickups are easy to miss as a strategic layer. | Tie pickups to builds, card telegraphy, and reward timing. |
| Rooms/shops | Exist as dungeon cards but can feel samey. | Make room/shop services create route and build decisions. |
| Objectives/tags | Rewards exist but are not always emotionally legible. | Make objective progress and completion feedback clearer. |

## Questions For Theory Passes
- Which systems should be removed, merged, or renamed?
- Which relics would still be exciting if their numbers were halved?
- Which mutators create a new way to think, not just worse conditions?
- Should talents exist as permanent profile choices, run starts, or relic-like drafts?
- Should every new action have a button, or should some be card-triggered only?
- How many card families can the game support before memory clarity suffers?
