# REF-031: Game over / run summary copy — clarity, a11y, and heading order
> **Rollup:** Addressed in codebase per [COMPLETION.md](./COMPLETION.md) (evidence row **031–040**).


## Problem

End-of-run screens carry high cognitive load; inconsistent headings, live regions, and button labels hurt screen-reader users and localization readiness.

## Proposed work

- Audit `GameOverScreen` (and related) for a single logical `<h1>`/`<h2>` order, button text that states outcome, and polite announcements for score changes.
- Extract user-visible strings to a single module or catalog if not already, for future i18n.
