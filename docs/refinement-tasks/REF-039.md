## Scope

Keep [`docs/FINDABLES.md`](../FINDABLES.md) aligned with the **authoritative** TypeScript definitions and spawn/claim logic so readers are not misled by older “proposal” wording.

## Authoritative code

| Location | Role |
|----------|------|
| [`src/shared/contracts.ts`](../../src/shared/contracts.ts) | `FindableKind` union; `FINDABLE_MATCH_SCORE`; `FINDABLE_MATCH_COMBO_SHARDS`; `Tile.findableKind`; `MutatorId` includes `'findables_floor'`. |
| [`src/shared/game.ts`](../../src/shared/game.ts) | `assignFindableKindsToTiles` (spawn counts, legacy gate, RNG seed string); `PICKUP_BASELINE_RULES_VERSION`; match/destroy paths clearing `findableKind` and applying rewards on **match** only. |
| [`src/renderer/hooks/useHudPoliteLiveAnnouncement.ts`](../../src/renderer/hooks/useHudPoliteLiveAnnouncement.ts) | `detectClaimedFindableKind`, announcement/toast copy keyed by `FindableKind`. |

## Drift addressed (snapshot)

- **Kinds:** Exactly `'shard_spark' \| 'score_glint'` — not an open-ended “e.g.” list.
- **Rewards:** Split between flat match score (`FINDABLE_MATCH_SCORE`) and immediate combo-shard gain (`FINDABLE_MATCH_COMBO_SHARDS`), not “pick one OR the other” at runtime.
- **Spawn / gate:** Post-baseline rules (`runRulesVersion >= PICKUP_BASELINE_RULES_VERSION`) are **not** “off unless mutator”; mutator `findables_floor` forces a **fuller** spawn profile (see doc + `assignFindableKindsToTiles`).
- **Destroy:** Implemented as **no findable claim** (destroy resolution does not run match scoring for findables).

## Recommended follow-ups (optional)

- If product wants “findables off by default” again, that is a **balance change** in `assignFindableKindsToTiles`, not only a doc fix.
