## Summary

Root **`tsconfig.json`** drives `yarn typecheck` (`tsc --noEmit`). **Type-aware ESLint** (`parserOptions.project: './tsconfig.json'`) applies only to an explicit `files` glob; everything else gets the shared **non–project** `@typescript-eslint` presets. Those three surfaces are **not the same footprint**: the notifications workspace and repo maintenance TypeScript can slip out of **`tsc`** and/or **type-aware lint** even though Vitest or `eslint .` still touches them.

## Authoritative layout today

### `tsconfig.json`

| `include` | Notes |
|-----------|--------|
| `src` | Full tree (renderer, shared, `main/`, `preload/`, tests under `src/**`) |
| `vite.config.mts`, `tsup.config.ts`, `vitest.setup.ts` | Root tooling entrypoints |

No top-level `exclude`. Path mapping: `@cross-repo-libs/notifications` → [`packages/notifications/src/index.ts`](../../packages/notifications/src/index.ts).

**What lands in the compiler program:** `tsc` follows imports from included roots. [`packages/notifications/src/`](../../packages/notifications/src/) **production** files used by the app are typechecked; **package unit tests** (e.g. `*.test.ts` under the package) are **not** pulled in unless referenced. **`scripts/run-mechanics-appendix.ts`** is **not** included (confirmed: `tsc --listFilesOnly` lists no `scripts/` paths). **`playwright.config.ts`** is **not** included.

### `eslint.config.mjs`

| Block | `files` | Type-aware (`project`) |
|--------|---------|-------------------------|
| Typed | `src/**/*.{ts,tsx}`, `vite.config.mts`, `tsup.config.ts`, `vitest.setup.ts` | `./tsconfig.json` — enables e.g. `@typescript-eslint/no-misused-promises` |
| E2E / Playwright | `e2e/**/*.ts`, `playwright.config.ts` | **Off** (`typescript-eslint` `disableTypeChecked`) |

**Not in the typed glob:** [`packages/notifications/src/**/*.{ts,tsx}`](../../packages/notifications/src/index.ts) — ESLint still runs (recommended rules), but **`parserOptions` stays empty** for those paths, so type-aware rules do not apply. Same for **`scripts/**/*.ts`** (only `scripts/**/*.mjs` / `*.cjs` are in `ignores`).

### Vitest (see [`vite.config.mts`](../../vite.config.mts))

`test.include` covers `src/**/*.{test,spec}.{ts,tsx}` and `packages/notifications/src/**/*.{test,spec}.{ts,tsx}` — **broader than** what root `tsc` necessarily typechecks for the package tests.

## Gaps (high signal)

| Area | `yarn typecheck` | Type-aware ESLint | Risk |
|------|------------------|-------------------|------|
| `src/**` | Yes | Yes | Aligned |
| Root configs (`vite` / `tsup` / `vitest.setup`) | Yes | Yes | Aligned |
| `packages/notifications/src` (app-imported) | Yes (via graph) | **No** (`parserOptions` not set for that path) | Weaker lint parity vs `src/`; same for `consistent-type-imports` / `no-misused-promises` |
| `packages/notifications/src/**/*.test.*` | **Often no** (not imported from app) | **No** | Tests run under Vitest but may miss `tsc` errors until test run or editor |
| `scripts/run-mechanics-appendix.ts` | **No** | **No** | Docs pipeline script has no `typecheck` gate |
| `e2e/**`, `playwright.config.ts` | **No** | **No** (by design in config) | Acceptable if team accepts Playwright-only typing; optional to tighten |

## Single refinement candidate

**Unify the “typed” triangle** in one small, explicit change set: (1) expand root `tsconfig.json` `include` so **package tests** and **`scripts/run-mechanics-appendix.ts`** (and optionally **`playwright.config.ts`**) participate in `yarn typecheck`; (2) extend ESLint’s typed `files` glob to **`packages/notifications/src/**/*.{ts,tsx}`** (and the same script path if included in `tsconfig`) so **`parserOptions.project`** applies consistently. Optionally add a **`references`** / second `tsconfig` only if composite projects are preferred over a flat include — avoid duplicating path mappings.

Keep the **e2e + `playwright.config.ts` `disableTypeChecked`** block unless you deliberately add Playwright config to `tsconfig` and want stricter lint there.
