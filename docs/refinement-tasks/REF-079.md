# REF-079: E2E test isolation — storage, clocks, and parallel workers
> **Rollup:** Addressed in codebase per [COMPLETION.md](./COMPLETION.md) (evidence row **071–080**).


## Problem

Playwright tests may share storage or time mocks, causing order-dependent failures.

## Proposed work

- Per-test storage sandbox (`storageState` / origin reset); fake timers policy documented.
- Shard e2e by feature folder with dedicated `test.describe.configure`.
