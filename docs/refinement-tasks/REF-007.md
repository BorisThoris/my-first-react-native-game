# REF-007: Single source of truth for `desktop:*` IPC channel strings (preload ↔ main)
> **Rollup:** Addressed in codebase per [COMPLETION.md](./COMPLETION.md) (evidence row **001–010**).


## Problem

`src/preload/index.ts` wires `DesktopApi` to `ipcRenderer.invoke('desktop:…')` using **literal channel strings**, and `src/main/ipc.ts` registers `ipcMain.handle` with the **same literals** again. The public contract is `DesktopApi` in `src/shared/contracts.ts`, but **channel names are not represented there**, so:

- Adding or renaming a desktop API requires matching edits in **two** places; a typo or drift fails only at **runtime** (invoke with no handler, or handler never called from preload).
- Code review has no compiler assist tying `exposeInMainWorld('desktop', …)` methods to `registerIpcHandlers` entries beyond human checking.

The preload script itself is intentionally minimal (Electron `contextBridge`); the refinement is **mechanical safety** around the boundary, not new surface area.

## Proposed work

1. Add a small shared module (e.g. **`src/shared/desktop-ipc-channels.ts`**) exporting a **`const` object** (or similar) whose values are the canonical `desktop:*` strings—one key per `DesktopApi` method in a consistent order (`getSettings`, `saveSettings`, …).
2. **`src/preload/index.ts`** — replace string literals with imports from that module for every `ipcRenderer.invoke(…)`.
3. **`src/main/ipc.ts`** — replace string literals in `ipcMain.handle(…)` with the **same** imports.

Optional hardening (same PR if cheap): a **satisfies** or helper type that ensures the channel map covers every key of a small `Record<keyof DesktopApi, string>`-shaped type so a new `DesktopApi` method forces updating the map at compile time.

## Acceptance

- No behavioral change: same IPC channels, same handler semantics, same `contextBridge` API.
- Unit tests and typecheck pass (`pnpm test` / `pnpm exec tsc` as used in CI).
- New desktop IPC in the future should be **one** string definition + contract + preload mapping + main handler, with the string defined only in the shared module.
