# REF-005 — WebGL face spec: replace fixed post-flip sleep with a condition wait

> **Rollup:** Addressed in codebase per [COMPLETION.md](./COMPLETION.md) (**001–010**). Post-flip capture uses a bounded stability poll — see Status.

**Category:** E2E (`e2e/`)  
**Gap type:** Flake risk (CI: `yarn test:e2e:renderer-qa`)  
**Priority:** P2

## Summary

[`e2e/tile-card-face-webgl.spec.ts`](../../e2e/tile-card-face-webgl.spec.ts) captures the board **before** and **after** one tile flip and asserts a bounded PNG diff ratio (QA-005). **After** the flip is applied (`readFrameHiddenTileCount` drops), the spec **does not** use a long fixed sleep. It **`expect.poll`s** until **two consecutive** stage-shell screenshots are **byte-identical** (short 100ms gap between captures inside the poll), with a **25s** timeout — i.e. a bounded wait for a **stable WebGL buffer** before the final “one flipped” shot.

A short **`waitForTimeout(400)`** remains **before** the pre-flip baseline screenshot (settle after canvas dimension poll). That is separate from the old post-flip 2200ms issue.

## Evidence

- [`e2e/tile-card-face-webgl.spec.ts`](../../e2e/tile-card-face-webgl.spec.ts) — `expect.poll` block after flip until paired PNGs match; then `shotFlipped` for diff vs `shotHidden`.
- [`e2e/README.md`](../../e2e/README.md) — related flake guidance for other scenarios (polling vs fixed sleeps).
- Root `package.json` — `test:e2e:renderer-qa` includes this file.

## Recommendation (historical)

The original ask was to replace a **fixed post-flip sleep** with a condition-driven wait. **Implemented** via consecutive-screenshot stability polling as above. Further hardening (e.g. replacing the pre-baseline 400ms) is optional.

## Scope / non-goals

- Do not widen the pixel-diff threshold (`maxDiffRatio`) unless product intentionally accepts more visual drift; this task is **timing robustness**, not loosening QA-005 bounds.
