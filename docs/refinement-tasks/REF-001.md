# REF-001: Unit-test `tilesArePairMatch` (wild / decoy / normal parity)
> **Rollup:** Addressed in codebase per [COMPLETION.md](./COMPLETION.md) (evidence row **001–010**).


## Problem

`tilesArePairMatch` in `src/shared/game.ts` is the single source of truth for whether two tiles count as a match; comments in the file tie it to gambit resolution and UI helpers (`tileResolvingSelection.ts`, `pairProximityHint.ts`). There are no direct Vitest assertions on this pure function, so a refactor or accidental change to decoy, wild, or same-`pairKey` behavior could desync `resolveBoardTurn` from on-screen match/mismatch feedback.

## Proposed work

- Add a `describe('tilesArePairMatch', …)` block in `game.test.ts` with small `Tile` fixtures (minimal fields; include `DECOY_PAIR_KEY` and `WILD_PAIR_KEY`).
- Assert: two normal tiles with the same `pairKey` match; any tile involving the decoy key does not match; wild (`WILD_PAIR_KEY`) with a non-wild real `pairKey` matches; document intended behavior for two wild tiles so it cannot change by accident.
- Optionally: mismatched normal `pairKey` returns false.
