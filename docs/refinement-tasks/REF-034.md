# REF-034: Main-process persistence — error surfaces and user-visible recovery
> **Rollup:** Addressed in codebase per [COMPLETION.md](./COMPLETION.md) (evidence row **031–040**).


## Problem

`src/main/persistence.ts` (and callers) write save data to disk. Silent failure or partial writes can corrupt runs without actionable UI.

## Proposed work

- Map failure modes (disk full, permission, EBUSY); ensure renderer receives structured errors.
- Add logging categories (no PII) and a user-facing toast or settings notice when autosave fails repeatedly.

## Files

| Action | Path |
|--------|------|
| Edit | `src/main/persistence.ts`, `src/main/index.ts`, renderer bridge |
| Tests | Unit tests with mocked `fs` where practical |

## Acceptance

- Failure paths are tested or documented; user never loses data silently without warning.

**reliability**

**P1**

## Status

**Done** — `PersistenceService` wraps `electron-store` writes with `PersistenceWriteError`; renderer uses `persistBridge` + `persistenceWriteNotice` on **Main menu**, **GameScreen**, and **Settings** (see [COMPLETION.md](./COMPLETION.md)).
