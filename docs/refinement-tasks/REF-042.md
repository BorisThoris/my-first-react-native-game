# REF-042: Card art `svgIds` — collision and import hygiene

**Status (2026-04):** The experimental `components/cards/cardArt/` React overlay tree was removed from the repo; this task no longer applies. Rollup evidence below predates removal.

> **Rollup:** Addressed in codebase per [COMPLETION.md](./COMPLETION.md) (evidence row **041–050**).


## Problem

`src/renderer/components/cards/cardArt/svgIds.ts` centralizes DOM ids for embedded SVG. Duplicate ids across cards break `url(#id)` references and a11y.

## Proposed work

- Add a dev/test scan that all registered ids are unique per sprite sheet (or per export batch).
- Align naming convention with `cardArt/index.ts` exports.
