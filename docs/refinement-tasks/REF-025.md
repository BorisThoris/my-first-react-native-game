# REF-025: `utc-countdown` boundary tests (DST, midnight, negative remaining)
> **Rollup:** Addressed in codebase per [COMPLETION.md](./COMPLETION.md) (evidence row **021–030**).


## Problem

`src/shared/utc-countdown.ts` formats time-until events. Off-by-one around UTC boundaries and DST transitions can show wrong values for limited-time UI without dedicated tests.

## Proposed work

- Table-driven tests with fixed `Date` inputs (use explicit UTC strings) for: zero remaining, negative (already passed), and a boundary near local midnight if any local formatting exists.
- Keep tests independent of machine local TZ where possible, or set `TZ` in test only if the module is locale-sensitive.
