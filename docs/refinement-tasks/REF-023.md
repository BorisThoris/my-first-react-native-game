# REF-023: `run-export` round-trip and version field discipline
> **Rollup:** Addressed in codebase per [COMPLETION.md](./COMPLETION.md) (evidence row **021–030**).


## Problem

`src/shared/run-export.ts` encodes run summaries for sharing or debugging. Format drift (extra fields, optional defaults) can produce exports that older readers mishandle or that round-trip with silent loss.

## Proposed work

- Add Vitest coverage for parse → serialize → parse identity on minimal and maximal fixture objects.
- Document (in file header or `contracts`) which fields are required for forward compatibility.
