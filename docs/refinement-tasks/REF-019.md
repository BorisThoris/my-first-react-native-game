# REF-019: Align Vitest setup docs with happy-dom `matchMedia` + `visualViewport` polyfill
> **Rollup:** Addressed in codebase per [COMPLETION.md](./COMPLETION.md) (evidence row **011–020**).


## Problem

`vitest.setup.ts` was refined to rely on happy-dom’s built-in `matchMedia` (desktop-like defaults) and to install a `visualViewport` polyfill so hooks that subscribe to `visualViewport` + `window` resize behave consistently in tests. [TOOLING.md](../internal-wiki/TOOLING.md) may still describe an explicit `matchMedia` mock—doc drift.

## Proposed work

- Update **TOOLING.md** Vitest subsection: describe happy-dom `matchMedia` defaults; state that tests needing coarse-pointer behavior override `window.matchMedia` locally.
- Document the `visualViewport` polyfill purpose (happy-dom gap) in TOOLING or a one-line comment pointer from README.
- Optional: add a short note in [APP_ANALYSIS.md](../internal-wiki/APP_ANALYSIS.md) under test infra if this becomes the canonical story.
