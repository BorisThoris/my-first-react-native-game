# REF-032 — Modal z-index ladder (OverlayModal vs Settings vs Meta)
> **Rollup:** Addressed in codebase per [COMPLETION.md](./COMPLETION.md) (evidence row **031–040**).


**Category:** UI / layering  
**Priority:** low (documentation)  
**Scope:** `OverlayModal.module.css`, `MetaScreen.module.css`, `SettingsScreen.module.css`, `App.tsx` (OVR-008)

## Observation

Several full-viewport shells share the same frosted-scrim vocabulary (`theme.ts` OVR-001/002/003) but must **never** compete at the same z-index. Conflicting values cause the wrong surface to receive clicks, paint order bugs, or confusing a11y stacking.

## Single ladder (canonical)

**Authoritative narrative:** `App.tsx`, block comment **OVR-008 / HUD-013** (keep in sync when numbers change).

For **in-run gameplay** overlays portaled or fixed above the game column, use this **strict low → high** order:

| z-index | Surface | CSS module | Typical use |
|--------:|---------|------------|---------------|
| **21** | `OverlayModal` backdrop | `OverlayModal.module.css` `.backdrop` | Pause, floor cleared, abandon, path-picker dialogs |
| **22** | Meta in-run modal shell | `MetaScreen.module.css` `.modalOverlay` | In-run inventory / codex (above gameplay `OverlayModal` so study UI wins when both concepts could overlap in the tree) |
| **24** | Settings in-run modal shell | `SettingsScreen.module.css` `.shellModal` | Run settings (must paint **above** 21/22 so the player can configure without dismissing underlying meta/inventory flows) |

**Rule:** New full-screen modal scrims in this family should extend the ladder (e.g. **26+**) rather than reusing **21**, **22**, or **24**.

## Related (not the same stack)

- **StartupIntro** (`StartupIntro.module.css` `.overlay`): portaled to `document.body` at **`z-index: 9000`**, above the gameplay modal ladder, so the intro sequence always wins while visible. Inner controls use **local** stacking (e.g. `.motionCta` at `z-index: 8` **inside** the intro overlay).
- **In-game column** (HUD, board, rails): separate subtree; see `GameScreen.module.css` and the **in-game column** subsection in the same `App.tsx` comment.
