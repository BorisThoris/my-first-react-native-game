# REF-009 — Tile board WebGL vs screen reader model
> **Rollup:** Addressed in codebase per [COMPLETION.md](./COMPLETION.md) (evidence row **001–010**).


**Category:** a11y  
**Priority:** medium  
**Scope:** `src/renderer/components/TileBoard.tsx`, `src/renderer/components/TileBoardScene.tsx`

## Observation

The playable tile grid is rendered in WebGL (`@react-three/fiber` `Canvas`) while keyboard operation and announcements are implemented on the surrounding DOM (`role="application"`, `aria-live="polite"`). The raw `<canvas>` element does not expose a useful accessibility tree for tiles; leaving it in the tab order / accessibility tree can add noise without benefit.

## Change (done)

- Set **`aria-hidden`** on the R3F `Canvas` in `TileBoard.tsx` so assistive technologies treat the bitmap surface as decorative. Interaction semantics remain on the `role="application"` container and the polite live region.
- Document the contract on **`focusedTileId`** in `TileBoardScene.tsx` (focus ring driven from parent; SR does not use the canvas subtree).
