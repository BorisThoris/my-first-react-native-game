## Summary

Main process wires **one** `BrowserWindow`, **single-instance** behavior, lazy `PersistenceService` + `SteamAdapter`, and **one-time** `registerIpcHandlers`. Persistence is a single `electron-store` document (`memory-dungeon-save`) keyed as `saveData`, always passed through `normalizeSaveData` from `src/shared/save-data.ts`. Preload exposes a typed `window.desktop` that mirrors IPC channels 1:1. Renderer consumption goes through `src/renderer/desktop-client.ts` (Electron) or **localStorage** fallback when `window.desktop` is absent.

**Canonical channel names** (and the `DesktopApi` → invoke map) live in [`src/shared/ipc-channels.ts`](../../src/shared/ipc-channels.ts) (`IPC_CHANNELS`: `save:*`, `steam:*`, `window:*`). [`ipc.ts`](../../src/main/ipc.ts) registers each handler under **both** the canonical name and the legacy `desktop:*` alias (`IPC_CHANNELS_LEGACY_DESKTOP`) so older call sites keep working. The table below uses the **legacy** string labels to match historical notes; map mentally to `IPC_CHANNELS` for new code.

## Current state (findings)

### Lifecycle (`index.ts`)

- `app.requestSingleInstanceLock()` — second instance focuses existing window via `createOrShowMainWindow`.
- `ensureServicesAndIpc()` constructs `PersistenceService` and `createSteamAdapter()`, then calls `registerIpcHandlers` **once** (`ipcHandlersRegistered`), passing `() => mainWindow` so IPC works after macOS reopen without re-registering handlers.
- Window loads Vite dev URL + detached DevTools when `VITE_DEV_SERVER_URL` is set; otherwise `dist/index.html`.
- `window-all-closed`: quits except on darwin (standard Electron pattern).

### IPC channel inventory (`ipc.ts` ↔ `preload/index.ts` ↔ `DesktopApi`)

| Channel | Main behavior |
|--------|----------------|
| `desktop:get-settings` | `persistence.getSettings()` |
| `desktop:get-save-data` | `persistence.getSaveData()` |
| `desktop:is-steam-connected` | `steamAdapter.isConnected()` |
| `desktop:set-display-mode` | `setFullScreen(mode === 'fullscreen')` on live window; **no-op + warn** if no window |
| `desktop:save-settings` | `persistence.saveSettings` then applies display mode to window if present |
| `desktop:save-game` | `persistence.saveGame` |
| `desktop:unlock-achievement` | **`persistence.unlockAchievement` first**, then `steamAdapter.unlockAchievement` (matches [APP_ANALYSIS.md](../internal-wiki/APP_ANALYSIS.md) note: local persist before Steam) |
| `desktop:quit-app` | `app.quit()` |

All handlers wrap errors with `console.error` and rethrow (invoke promise rejects on failure).

### Persistence (`persistence.ts`)

- `electron-store` `Store<StoreShape>` with `defaults: { saveData: normalizeSaveData() }`.
- Every read uses `normalizeSaveData(this.store.get('saveData'))`; writes use `normalizeSaveData` before `store.set`.
- Achievements merged via spread over existing `achievements` object in `unlockAchievement`.

### Steam (`steam.ts`, IPC-adjacent)

- Init from `STEAM_APP_ID` when set; `electronEnableSteamOverlay` when available.
- Failure → mock adapter (`isConnected` false, unlock no-ops). Achievements still recorded in disk save via persistence.

## Risks / gaps (observations, not a backlog)

- **No automated tests** under `src/main/` (Vitest does not cover `PersistenceService` or IPC wiring).
- **`desktop:set-display-mode`** when the window is missing only logs — renderer may assume fullscreen state; low frequency (race on shutdown).
- **Dev hot-reload**: handlers register once per process; typical for Electron, but duplicate registration attempts are prevented by flag (not an issue unless `registerIpcHandlers` is refactored to be callable twice).

## Recommended refinement (one task)

**Add focused Vitest coverage for `PersistenceService`** using a mocked `electron-store` (or injecting a store factory for testability), asserting: default shape matches `createDefaultSaveData()` / `normalizeSaveData`; `saveSettings` / `saveGame` / `unlockAchievement` persist normalized data and preserve achievement merges. Keep tests **pure main-layer** (no real IPC), so CI validates persistence invariants without launching Electron.
