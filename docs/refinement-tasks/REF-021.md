# REF-021: Expand `pairProximityHint` tests for board edges and blocked paths
> **Rollup:** Addressed in codebase per [COMPLETION.md](./COMPLETION.md) (evidence row **021–030**).


## Problem

`src/shared/pairProximityHint.ts` picks a hint pair for proximity-based assists. `pairProximityHint.test.ts` exists but edge-heavy boards (corners, fully blocked adjacency, empty eligible sets) are easy to regress when tuning heuristics or performance caps.

## Proposed work

- Add focused cases: single valid pair on last row; no legal pair (document expected return); board where the closest pair is not the first in scan order.
- If the module exposes internal ordering, lock it with explicit fixtures rather than snapshot noise.
