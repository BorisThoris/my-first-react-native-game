# REF-033: Preload script surface — IPC allowlist and type parity with renderer
> **Rollup:** Addressed in codebase per [COMPLETION.md](./COMPLETION.md) (evidence row **031–040**).


## Problem

`src/preload/index.ts` exposes APIs to the renderer. Any mismatch with `src/main/ipc.ts` channel names or payload shapes is a security and stability risk.

## Proposed work

- Generate or manually verify a single source of truth for channel names (const shared module used by main + preload).
- Add tests or type-bridge so adding a channel requires updating all three layers.

## Files

| Action | Path |
|--------|------|
| Edit | `src/preload/index.ts`, `src/main/ipc.ts`, `src/renderer/desktop.d.ts` |
| Optional shared | `src/shared/ipc-channels.ts` (new) |

## Acceptance

- Typecheck fails if preload exposes a method main does not handle; documented security assumptions (no arbitrary node access).
