# REF-093: Test file naming — `.test.ts` vs `.test.tsx` convention
> **Rollup:** Addressed in codebase per [COMPLETION.md](./COMPLETION.md) (evidence row **091–099**).


## Problem

Mixed extensions for React tests confuse tooling (Vitest include globs, coverage).

## Proposed work

- Rule: any file importing JSX uses `.tsx`; pure logic uses `.ts`.
- ESLint or script to flag mismatches; rename outliers (e.g. `useFitShellZoom`).

## Files

| Action | Path |

| Edit | eslint config, rename tests as needed |

## Acceptance

- Globs stay simple; no parse errors from `.ts` files containing JSX.
