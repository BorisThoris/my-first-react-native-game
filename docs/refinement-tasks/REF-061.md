# REF-061: `toolbarRoving` vs modal focus trap — no double roving tab order
> **Rollup:** Addressed in codebase per [COMPLETION.md](./COMPLETION.md) (evidence row **061–070**).


## Problem

`src/renderer/a11y/toolbarRoving.ts` implements roving tabindex. When a modal opens, focus trap and toolbar roving can conflict.

## Proposed work

- Define pause/resume API for roving when `OverlayModal` mounts.
- RTL tests: Tab cycles only within modal when open.
