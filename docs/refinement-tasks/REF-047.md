# REF-047: `tileBoardViewport` resize debounce and DPR changes
> **Rollup:** Addressed in codebase per [COMPLETION.md](./COMPLETION.md) (evidence row **041–050**).


## Problem

`src/renderer/components/tileBoardViewport.ts` maps screen rects to board space. Rapid resize / `devicePixelRatio` changes can thrash layout or WebGL resize.

## Proposed work

- Consolidate resize listeners; use `ResizeObserver` where available; debounce or rAF-coalesce.
- Tests for stable pick coordinates across a simulated resize sequence.
