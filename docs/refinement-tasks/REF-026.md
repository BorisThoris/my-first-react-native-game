# REF-026 — `theme.ts` token drift vs CSS modules

| Field | Value |
|--------|--------|
| **ID** | REF-026 |

## Summary

**Authoritative tokens** live in [`theme.ts`](../../src/renderer/styles/theme.ts): `RENDERER_THEME.colors` (TS consumers: Canvas, Skia, textures) and `RENDERER_THEME.cssVars` (CSS custom properties). At boot, [`main.tsx`](../../src/renderer/main.tsx) applies **all** `cssVars` to `document.documentElement`. [`App.tsx`](../../src/renderer/App.tsx) spreads the same map via `buildRendererThemeStyle()` onto the `.app` root, plus density / `--ui-scale` / reduced-motion overrides.

**Drift** happens when `.module.css` (or `global.css`) hardcodes hex/rgba stacks that duplicate or diverge from those tokens, or when `var(--theme-*, fallback)` uses literals that **do not match** [`theme.ts`](../../src/renderer/styles/theme.ts).

## Drift categories

| Category | Example | Risk |
|----------|---------|------|
| **A — Duplicate literals** | Repeated `rgba(255, 243, 217, 0.05)` gloss lines across HUD/meta shells | Palette tweak requires many file edits; visual inconsistency if one file is missed |
| **B — `var(..., fallback)` mismatch** | [`App.module.css`](../../src/renderer/styles/App.module.css) `.skipLink`: `--theme-void` fallback `#0b0d12` vs canonical `#05050a` | Broken or pre-bootstrap styles look “off brand” |
| **C — Semantic color without token** | `#8c62df` in [`OverlayModal.module.css`](../../src/renderer/components/OverlayModal.module.css) — matches `colors.violet` / `--theme-violet` | Same as A; harder to grep than rgba stacks |
| **D — TS vs CSS parity** | `RENDERER_THEME.colors.*` must stay aligned with matching `--theme-*` entries (today maintained by hand) | Renaming a hex in one object but not the other |

## High-signal locations (non-exhaustive)

| Area | Notes |
|------|--------|
| **HUD chrome** | [`GameScreen.module.css`](../../src/renderer/components/GameScreen.module.css) — many `rgba(255, 243, 217, …)` stacks; [`theme.ts`](../../src/renderer/styles/theme.ts) documents HUD-specific vars under `HUD-*` comments — prefer `var(--theme-hud-*)` where the token exists |
| **Meta / settings / codex / inventory** | Shared “plate” gloss patterns (`rgba(255, 232, 190, …)`, `rgba(255, 243, 217, …)`) overlap [`--theme-meta-*`](../../src/renderer/styles/theme.ts) and panel surfaces |
| **OverlayModal** | Mix of `color-mix`, `#8c62df`, `rgba(87, 220, 255, …)` — accent RGB differs from `--theme-cyan` / `--theme-violet` numeric bases; intentional motion variants vs drift should be documented in-file |
| **ChooseYourPathScreen** | Several `var(--theme-text, #e8ecf4)`-style fallbacks — `#e8ecf4` ≠ `--theme-text` `#f4ecdc` |
| **StartupIntro / MainMenu** | Atmospheric gradients duplicate bloom vocabulary also expressed as `--theme-intro-*` / `--theme-shell-*` |
| **Dev sandbox** | [`MatchedCardRimFireSandbox.module.css`](../../src/renderer/dev/MatchedCardRimFireSandbox.module.css) — ad hoc palette; lowest priority |

## Recommended refinements (incremental)

1. **Fix category B first** — Any `var(--theme-*, …)` fallback in checked-in CSS should use the **exact** string from [`RENDERER_THEME.cssVars`](../../src/renderer/styles/theme.ts) (or drop the fallback if `main.tsx` + `.app` always supply vars).
2. **Category A — mechanical passes** — Replace literals that already have a named `--theme-*` or component-local token with `var(…)`. Prefer adding a missing token in [`theme.ts`](../../src/renderer/styles/theme.ts) over a third copy of the same gradient in a new file.
3. **Category D — optional guard** — A small unit test (or script) that asserts `colors` entries match their `--theme-*` counterparts for the same semantic name would catch split-brain edits; only worth it if drift recurs in review.
