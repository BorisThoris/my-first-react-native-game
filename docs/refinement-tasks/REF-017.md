## Summary

Telemetry is a **privacy-first, injectable sink** (K1): `trackEvent` is a **no-op** until `setTelemetrySink` assigns a handler. **In-repo wiring:** only **`App.tsx`** installs a sink, and **only in dev** (`console.debug`). **Production** has **no** bundled Plausible/analytics sink—hosts must inject one. **All product events** today live in **`useAppStore.ts`**: two event names, **`run_start`** and **`run_complete`**, both tied to run lifecycle.

## API surface

```1:18:src/shared/telemetry.ts
/**
 * Privacy-first telemetry hook (K1). Default is no-op; host can install a sink (e.g. Plausible) without changing call sites.
 */
export type TelemetryPayload = Record<string, string | number | boolean | null | undefined>;

export type TelemetrySink = (event: string, payload: TelemetryPayload) => void;

let sink: TelemetrySink | null = null;

export const setTelemetrySink = (next: TelemetrySink | null): void => {
    sink = next;
};

export const trackEvent = (event: string, payload: TelemetryPayload = {}): void => {
    if (sink) {
        sink(event, payload);
    }
};
```

- **Payload shape:** flat key/value; values exclude objects/arrays (enforced by type only at compile time).
- **Threading:** synchronous invoke; no batching, sampling, or error isolation—sink exceptions would propagate to callers.

## Event catalog (actual call sites)

### `run_complete`

- **When:** `applyResolvedRun` path when run transitions to **`gameOver`** and `lastRunSummary` exists.
- **Payload keys:** `mode`, `practice`, `highestLevel`, `totalScore`, `mutatorCount`, `relicCount`.

### `run_start`

- **When:** Each store action that begins a run (standard, daily, gauntlet, puzzle builtin/import, practice, scholar contract, meditation ± mutators, pin vow, wild, import-from-clipboard) and **`restartRun`**.
- **Common keys:** `mode`, `practice` (always).
- **Variant flags / extra keys:** `puzzleId`, `puzzleImport` (imported puzzle JSON); `scholar`; `meditation_focus_count`, `meditation_focus` (comma-joined mutator ids); `pinVow`; `wild`; `imported` (run import); `restarted: true`.

**Count:** 1× `run_complete`, 14× `run_start` (grep-level; several branches share the same payload shape).

## Gaps and candidate refinements (not prescriptions)

1. **Production sink:** No in-repo analytics backend; align with product on Electron/host injection vs bundled SDK.
2. **Schema discipline:** Event names are stringly-typed; payloads are not discriminated by `event`—consider a typed map or shared contract for sink implementers.
3. **Naming consistency:** `meditation_focus_count` / `meditation_focus` use **snake_case**; other keys are **camelCase**.
4. **Privacy review:** `meditation_focus` exposes mutator ids; `puzzleId` may be synthetic (`import:…`) or builtin id—confirm acceptable for any real sink.
5. **DRY:** Many identical `trackEvent('run_start', { mode, practice })` lines—optional helper `trackRunStart(run, extras?)` to reduce drift.
6. **Resilience:** Optional `try/catch` inside `trackEvent` so a broken sink cannot break gameplay store paths.
