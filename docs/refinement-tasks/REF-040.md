## Summary

[`docs/BALANCE_NOTES.md`](../BALANCE_NOTES.md) states that balance constants live in **`contracts.ts`** “unless noted,” but the same file also cites **symbol band** thresholds (`tile-symbol-catalog.ts`), **presentation mutator match penalties** (`game.ts`), and **gauntlet menu presets** (`run-mode-catalog.ts` / `createGauntletRun` default). Inline numbers (tables and prose) can drift when tuning changes land without a doc pass.

Goal: a **small, automatable drift check** (CI-friendly) that compares **documented numeric claims** in `BALANCE_NOTES.md` to **authoritative exports or literals** in code—similar in spirit to [`mechanics-catalog-appendix-builder.ts`](../../src/shared/mechanics-catalog-appendix-builder.ts) + `yarn docs:mechanics-appendix`, but **read-only** (fail the check rather than silently rewriting prose).

## Authoritative sources today (high signal)

| Topic | Code | What the doc currently echoes |
|------|------|--------------------------------|
| Core tuning | `contracts.ts` | `INITIAL_LIVES`, `MAX_LIVES`, memorize timings, shuffle/destroy banks, `GAME_RULES_VERSION`, score bonuses, etc. |
| Symbol bands | `tile-symbol-catalog.ts` | `SYMBOL_BAND_LAST_LEVEL_NUMERIC`, `SYMBOL_BAND_LAST_LEVEL_LETTER` (table shows parenthetical levels) |
| Presentation penalties | `game.ts` | `PRESENTATION_MUTATOR_PENALTY_*` for `wide_recall` / `silhouette_twist` / `distraction_channel` (doc table) |
| Gauntlet lengths | `run-mode-catalog.ts`, `game.ts` `createGauntletRun`, `useAppStore` defaults | Menu **5 / 10 / 15** minutes; factory default **10m** |

## Script idea (recommended shape)

1. **Prefer imports over regex-parsing TypeScript.** Add a thin module (e.g. `src/shared/balance-notes-snapshot.ts` or under `scripts/`) that **re-exports or copies** the few values the doc cares about:
   - `import * as contracts from './contracts'` and pick `INITIAL_LIVES`, `MAX_LIVES`, `MEMORIZE_*`, `INITIAL_SHUFFLE_CHARGES`, `GAME_RULES_VERSION`, …
   - `SYMBOL_BAND_*` from `tile-symbol-catalog.ts`
   - Either **export** the three `PRESENTATION_MUTATOR_PENALTY_*` constants from `game.ts` (small refactor) or duplicate the numbers in the snapshot module **only** if you want zero churn in `game.ts` (worse—prefer export)
   - Gauntlet: read `durationMs` from `run-mode-catalog` preset entries and `createGauntletRun` default (ms → minutes for display)

2. **Parse `docs/BALANCE_NOTES.md` minimally.** Target only **stable anchors**:
   - Tables (symbol band, presentation penalties)—match row label → expected number
   - Optional: fenced block `<!-- balance-snapshot:check -->` … `<!-- /balance-snapshot -->` with **machine-generated** lines (future: `yarn docs:balance-notes` emits that block; drift script compares block to imports)

3. **Runner.** `tsx scripts/check-balance-notes-drift.ts` (or a Vitest file under `src/shared/*.test.ts`):
   - Exit **0** when doc numbers match snapshot
   - Exit **1** with a **diff-style** message: constant name, expected, found in doc (line number)

4. **Wire-up.** Add `yarn docs:balance-check` (or `yarn test:balance-notes`) and optionally run it in CI next to `yarn test` / `yarn typecheck`.

## Non-goals

- Fully validating free-form prose (“floor 6”, “Scholar”)—only **numeric / named constant** claims that map 1:1 to code
- Auto-editing narrative sections without an explicit “generated block” convention
