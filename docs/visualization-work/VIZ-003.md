# VIZ-003: Fixture / taxonomy for regression `pairKey`s

## Problem

The illustration regression set should **stress** distinct string shapes and hash behavior (tarot-style ids, shard/sigil names, numeric pairs). Without curation, new rules can ship without clear **which keys** must stay bit-stable.

## Proposed work

- Map current keys to categories (tarot / alpha-beta / numeric / long tokens); document gaps.
- Add keys deliberately to catch regressions in [`hashPairKey`](../../src/shared/hashPairKey.ts) sensitivity or illustration roll boundaries.
- Keep the JSON fixture as the single source of truth for “which keys we care about” unless code introduces a shared exported array.
