## Status

**Done** — manifest fields + troubleshooting + CI pointer in [README.md](./README.md) § Offline bake output / Troubleshooting.

## Problem

**`yarn bake:procedural-set`** produces local artifacts under `output/`; contributors need a clear **contract** for `manifest.json`, whether CI runs the bake, and how to avoid flaky runs when Vite HMR is active.

## Proposed work

- Document **manifest** fields next to PNGs (source of truth: writer in [`bake-procedural-illustration-set.ts`](../../scripts/bake-procedural-illustration-set.ts)).
- Decide: **CI job** (slow, brittle) vs **developer-only** bake on demand; if CI, define timeout, shard, or scheduled job.
- Keep script comments accurate on **navigation / HMR**: batch runs should use a **fresh document** per iteration when reusing a dev server so Playwright’s execution context is not torn down mid-loop.
