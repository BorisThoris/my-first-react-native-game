# VIZ-001: Documentation parity when version knobs change

## Status

**Done** — PR checklist in [ARCHITECTURE.md](./ARCHITECTURE.md) § **PR checklist (version bumps)**.

## Problem

Procedural illustration caches and Playwright fixtures depend on **`ILLUSTRATION_GEN_SCHEMA_VERSION`** and **`GAMEPLAY_CARD_VISUALS.textureVersion`**. Bumping those without updating [ARCHITECTURE.md](./ARCHITECTURE.md), PR notes, or regression fixtures creates **orphan changes** that are hard to audit.

## Proposed work

- Treat [ARCHITECTURE.md](./ARCHITECTURE.md) as the narrative source of truth for flows; bump or annotate it whenever shipping changes invalidate caches or fixtures.
- When bumping [`ILLUSTRATION_GEN_SCHEMA_VERSION`](../../src/renderer/cardFace/proceduralIllustration/illustrationSchemaVersion.ts), confirm ARCHITECTURE still names the correct files and knobs; note the bump in the PR description.
- When bumping [`GAMEPLAY_CARD_VISUALS.textureVersion`](../../src/renderer/components/gameplayVisualConfig.ts), do the same for overlay/tile integration.
- After intentional visual changes, run **`yarn regenerate:illustration-regression`** and commit fixture updates; reference this task ID in the PR.
